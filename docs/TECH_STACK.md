# 아이쑥크림(icecream) 기술 스택

> 유아 국민체력100 성장관리 서비스

## 1. 문서 목적

이 문서는 아이쑥크림의 초기 개발을 위한 기술 선택과 기본 원칙을 정리한다. 현재는 개발 착수 전 단계이므로, 아래 내용은 **MVP 구현을 위한 제안안**이다. 실제 구현 과정에서 팀의 숙련도와 운영 환경을 확인한 뒤 세부 버전을 확정한다.

- 프로젝트 상태: 개발 착수 전
- 제품 형태: 모바일 우선 반응형 웹
- 핵심 방향: FastAPI 기반 백엔드 + React 기반 프론트엔드
- 주요 기능: 유아 체력 진단, 측정 기록 저장, 성장 추적, 체력인증센터 안내, 활동 추천

## 2. 기술 선택 원칙

1. MVP에서는 개발 속도와 유지보수성을 우선한다.
2. 체력 등급 판정은 외부 API가 아닌 내부의 버전 관리되는 기준표를 사용한다.
3. 공공데이터 API 장애가 진단·기록·추적 기능으로 전파되지 않도록 외부 연동을 분리한다.
4. 아이와 보호자의 개인정보를 최소한으로 저장하고, 운영 환경에서는 비밀정보를 코드에 포함하지 않는다.
5. 모바일 화면을 기준으로 구현하되, 데스크톱에서도 사용할 수 있는 반응형 UI를 제공한다.
6. 초기에는 단순한 구조로 시작하고, 실제 병목이 확인된 뒤 캐시·비동기 작업·인프라를 확장한다.

## 3. 권장 기술 스택

### 3.1 전체 구성

| 영역 | 권장 기술 | 적용 목적 |
| :--- | :--- | :--- |
| 프론트엔드 | React + TypeScript + Vite | 반응형 웹 UI와 빠른 개발 환경 |
| UI 스타일 | Tailwind CSS | 모바일 우선 화면과 디자인 토큰 관리 |
| 아이콘 | Material Symbols Rounded | 디자인 명세와 일치하는 둥근 아이콘 |
| 한글 폰트 | Pretendard | 모바일 화면의 한글 가독성과 일관된 UI 문구 |
| 클라이언트 상태 | TanStack Query | 서버 상태 조회·캐싱·재요청 관리 |
| 클라이언트 라우팅 | React Router | 로그인·자녀·진단·기록 화면 라우팅 |
| 차트 | Recharts | 항목별 성장 추이 시각화 |
| 백엔드 | Python 3.12 + FastAPI | 타입 기반 비동기 API 서버 |
| API 스키마 | Pydantic v2 | 요청·응답 검증 및 문서화 |
| ORM | SQLAlchemy 2.x | 데이터베이스 모델과 쿼리 관리 |
| 데이터베이스 | PostgreSQL | 회원·자녀·측정 기록·공공데이터 저장 |
| 마이그레이션 | Alembic | 스키마 변경 이력 관리 |
| 인증 | 이메일/비밀번호 + JWT | PRD의 이메일 기반 회원 기능 구현 |
| 비밀번호 보안 | Argon2id 또는 bcrypt | 비밀번호 단방향 해시 저장 |
| 외부 연동 | `httpx` | 공공데이터포털 API 호출 |
| 테스트 | Pytest / Vitest / React Testing Library | 백엔드·프론트엔드 검증 |
| 코드 품질 | Ruff / mypy / ESLint / Prettier | 정적 검사와 일관된 포맷 (`npm run format`, `npm run format:check`) |
| 개발 환경 | Docker Compose | 로컬 PostgreSQL 등 개발환경 표준화 |
| 배포 | Docker 기반 클라우드 환경 | 배포 플랫폼에 종속되지 않는 운영 기반 |

### 3.2 초기에는 선택하지 않는 기술

- Redis: API 캐시나 백그라운드 작업이 실제로 필요해질 때 도입한다.
- Celery/RQ: 공공데이터 정기 동기화 작업이 생길 때 검토한다. MVP에서는 별도 관리 명령 또는 단순 스케줄러로 시작할 수 있다.
- 상태관리 라이브러리(Redux 등): 서버 상태는 TanStack Query로 관리하고, 작은 UI 상태는 React 기본 기능으로 처리한다.
- 네이티브 앱: 이번 MVP 범위는 반응형 웹이므로 제외한다.

## 4. 프론트엔드 화면 구현 규칙

- 아이콘은 `material-symbols` 패키지의 Material Symbols Rounded를 공통 `Icon` 컴포넌트에서 매핑해 사용한다. 이모지를 UI 아이콘으로 사용하지 않는다.
- 한글 본문·라벨·헤드라인은 Pretendard를 기본으로 사용한다.
- 긴 모바일 콘텐츠는 세로 스크롤을 허용하되, 하단 탭 바는 고정하고 헤더는 sticky 동작을 사용한다.
- 모든 아이콘 버튼에는 명확한 `aria-label`을 제공하고, 키보드 `focus-visible` 상태를 표시한다.
- 인터랙티브 요소의 최소 터치 영역은 44px로 유지하고, safe-area inset을 하단 내비게이션에 반영한다.
- `npm run format`과 `npm run format:check`으로 Prettier 포맷을 관리한다.
