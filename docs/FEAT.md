# 아이쑥크림(icecream) 기능명세서

> 유아기(만 4~6세) 자녀의 체력을 국민체력100 공식 기준으로 진단·연계·추적하는 서비스의 기능 상세 명세.
> PRD의 기능 ID 체계, 데이터베이스 설계, 판정 로직(기준표 JSON)을 통합한 개발 참조 문서.

## 1. 공통 규칙

### 1.1 인증
- 이메일 + 비밀번호 로그인, JWT(Access) 기반 인증.
- 비밀번호는 Argon2id/bcrypt 단방향 해시로 저장.
- 인증이 필요한 모든 API는 `Authorization: Bearer <token>` 요구. 누락·만료 시 `401`.
- 자녀·측정 등 소유 리소스는 토큰의 parent와 소유자가 일치하지 않으면 `403`.

### 1.2 식별자
- 외부 응답 ID는 `{resource}_{id}` 형식(예: `child_123`). API 경계에서 인코딩/디코딩, 내부는 bigint.

### 1.3 공통 응답 규칙
| 상황 | 코드 | 처리 |
| :--- | :--- | :--- |
| 성공(조회/수정) | 200 | 데이터 반환 |
| 성공(생성) | 201 | 생성 리소스 반환 |
| 입력 검증 실패 | 422 | Pydantic 검증 오류 상세 |
| 인증 실패 | 401 | 재로그인 안내 |
| 권한 없음 | 403 | 접근 불가 |
| 리소스 없음 | 404 | |
| 외부 API 장애 | 200(부분) 또는 503 | 핵심 기능 유지, 해당 섹션만 저하 |

## 2. 회원 및 자녀 프로필 (ACC)

### ACC-01 학부모 회원 가입/로그인
| 항목 | 내용 |
| :--- | :--- |
| 우선순위 | P0 |
| 설명 | 이메일 기반 가입·로그인. 한 계정에 다수 자녀 등록 가능. |
| 입력 | email, password |
| 처리 | 가입 시 이메일 중복 검사 → 비밀번호 해시 저장. 로그인 시 해시 검증 → JWT 발급. |
| 예외 | 이메일 중복(409), 비밀번호 정책 위반(422), 로그인 실패(401) |
| 인수 조건 | 로그인한 학부모가 자녀 2명 이상을 등록하고 프로필을 전환하며 조회할 수 있다. |

### ACC-02 자녀 프로필 등록
| 항목 | 내용 |
| :--- | :--- |
| 우선순위 | P0 |
| 설명 | 자녀 등록·수정·삭제. 진단 기준표 매칭에 사용되는 성별·개월수 관리. |
| 입력 | nickname, gender(male/female), birth_year_month |
| 처리 | 생년월로 개월수 자동 계산. 2019년 이전 또는 현재 연도 이상은 등록을 제한하고, 48~83개월 범위 밖이면 "유아기 측정 대상 범위 밖" 안내. |
| 규칙 | 개월수는 저장하지 않고 조회/측정 시점에 계산(시간에 따라 변함). |
| 예외 | 필수값 누락(422), 타 학부모 자녀 접근(403) |
| 인수 조건 | 생년월 입력 시 개월수가 정확히 계산되고, 범위를 벗어난 경우 안내 문구가 노출된다. |

## 3. 체력 진단 (DIA) — 핵심

### DIA-01 측정값 입력
| 항목 | 내용 |
| :--- | :--- |
| 우선순위 | P0 |
| 설명 | 유아기 8개 항목값과 측정 유형(정식/자가)을 입력받아 측정 세션 생성. |
| 입력 | child_id, type(official/self), measured_at, items[{item_key, value}] (일부만 허용) |
| 처리 | measurement + measurement_item(long) 생성. 자가측정은 center_id NULL. 측정 시점 개월수를 age_months_at_measure에 스냅샷 저장. |
| 규칙 | 미입력 항목은 measurement_item 행을 생성하지 않음(행 부재 = 미입력). |
| 예외 | 잘못된 item_key(422), 존재하지 않는 child(404) |
| 인수 조건 | 항목별 단위가 표시되고, 일부 항목만 입력해도 저장되며 미입력 항목은 판정에서 제외된다. |

### DIA-02 공식 기준표 기반 등급 판정
| 항목 | 내용 |
| :--- | :--- |
| 우선순위 | P0(핵심) |
| 설명 | 측정값을 국민체력100 유아기 기준표(JSON)에 대입해 종합 등급과 항목별 등급 산출. |
| 데이터 | 앱 메모리 로드된 기준표 JSON(성별×연령구간×등급×항목). |
| 판정 순서 | gradeOrder(fruit→flower→sprout) 순으로 각 등급 requiredItems 전부 충족하는 첫 등급 부여. 미충족 시 seed. |
| 등급별 평가 항목 | 열매·꽃 = 건강체력4 + 운동체력3(7개 전부) / 새싹 = 건강체력4 + BMI(5개) / 씨앗 = 폴백. |
| 방향성 비교 | 클수록 우수: `값 >= 기준` / 시간(민첩성·협응력): `값 <= 기준` / BMI: `값 < 기준`(strict). |
| 저장 | 종합 등급을 measurement.grade에, 항목별 등급을 measurement_item.item_grade에 보존(측정 당시 기준). |
| 인수 조건 | 남아 60~65개월의 임의 측정값 세트에 대해 종합 등급·항목별 판정이 기준표와 정확히 일치한다. |

### DIA-03 항목별 강·약 프로파일
| 항목 | 내용 |
| :--- | :--- |
| 우선순위 | P0 |
| 설명 | 판정 결과를 항목별로 시각화하고 강점/약점을 구분해 표시. |
| 처리 | item_grade 기준으로 강점(상위 등급)·약점(하위 등급) 분류. 약점 항목은 활동 추천(ACT-01) 진입점 제공. |
| 인수 조건 | 특정 항목이 하위 등급일 때 '개선 필요' 표식과 활동 추천 이동 링크가 노출된다. |

### DIA-04 판정 엔진 예외 처리
| 항목 | 내용 |
| :--- | :--- |
| 우선순위 | P0 |
| 설명 | 등급별 required_items 분리로 "값 없음 → 판정 불가" 버그 방지. |
| 규칙 | 새싹 판정에는 운동체력(순발력·민첩성·협응력) 값이 관여하지 않으므로, 해당 값이 없어도 새싹/씨앗 판정이 가능해야 함. |
| 규칙 | 특정 등급의 required_items 중 미입력 값이 있으면 그 등급은 판정 불가(다음 등급으로), seed는 항상 폴백. |
| 인수 조건 | 운동체력 값 없이 건강체력4+BMI만 입력해도 새싹/씨앗 판정이 정상 반환된다. |

## 11. MVP 데모 핵심 시나리오
> 심사에서 반드시 끊김 없이 시연할 흐름:
> 자녀 등록(ACC-02) → 측정값 입력(DIA-01) → 공식 등급 판정·프로파일(DIA-02/03) → 약점 활동 추천(ACT-01) → 근처 센터 연계(CTR-01) → 기록 저장·추이(REC-01/02)

우선 구현 순서: DIA(판정 엔진) → CTR·ACT(공공 API 연동) → REC(기록·추적) → INS(지역 인사이트) → SLF(자가측정).
