## 기획 전체 구조

### 서비스 정의
- 우리 아이(유아기 만 4~6세)의 체력을 국민체력100 공식 기준으로 판정해 '씨앗·새싹·꽃·열매' 등급과 항목별 강약을 알려주고, 근처 체력인증센터에서 무료 정식 측정을 받도록 연계하며, 측정 결과를 기록해 성장 추이를 추적하는 유아 체력관리 서비스

### 고객 가치 제안
- 학부모에게: 병원(키·몸무게)이나 어린이집에서 알 수 없던 아이의 체력 수준을 국가 공인 기준으로 확인해줍니다
- 인증서가 발급되지 않아 방치되기 쉬운 유아기 체력측정 결과지를, 등급 판정과 항목별 해석으로 의미 있게 관리해줍니다
- 집에서 간이 자가측정(참고용)으로 대략적 신호를 먼저 파악하고, 정식 측정이 필요하면 무료로 받을 수 있는 근처 센터를 바로 안내받습니다
- 측정 기록이 쌓여 아이 체력의 성장 추이와 약한 부분의 개선 여부를 한눈에 확인합니다
- 진단에서 끝내지 않고 약한 체력 요소별 가정 활동(공단 운동처방 콘텐츠)을 추천받아 무엇을 해야 할지 알게 됩니다
- 우리 동네의 유아 체력측정 참여 수준을 전국과 비교해 보여주어, 무료 측정 기회를 놓치지 않도록 안내합니다

### 차별성
- 계측이 아닌 체력을 다룸: 기존 육아 앱(아이키기록, 아기성장발달계산기 등)은 키·몸무게·머리둘레 등 신체 계측 기반. 근력·순발력·유연성 등 체력 요소를 국가 기준으로 판정하는 소비자 서비스는 부재
- 국민체력100 생태계 밀착: 뉴트리프렌즈 등 유아 성장관리 앱은 영양·계측 중심. 국민체력100의 체력측정 항목·공식 등급(씨앗~열매)·인증센터를 축으로 삼는 서비스는 없음. 주최 기관 데이터에 정확히 밀착한다는 점에서 대체 불가
- 방치된 결과지의 사후관리: 유아기 측정은 참고용이라 인증서 없이 결과지만 제공되어 관리가 끊김. 이 결과지를 받아 등급 해석·누적·추적해주는 서비스가 없음
- 공단과 경쟁이 아닌 보완: 공단 자가측정은 성인 대상 설문형(PAR-Q·IPAQ)이라 유아와 무관. 우리는 측정을 대신하지 않고 측정 이후(관리)와 측정으로 가는 길(센터 연계)을 맡아 센터 유입을 늘림
- 데이터를 소비하지 않고 통찰을 생산: 단순 기준표 대입·검색에 그치지 않고, 지역별 측정건수를 전국과 비교해 지역 참여 인사이트를 도출. 진단에서 활동 추천·재측정 유도까지 개선 루프를 완성

### 수익모델 정의
- 공모전·공공서비스 성격상 직접 과금은 하지 않는 것을 기본으로 함 (수혜자가 학부모·아동이라 유료화 시 공공성 훼손)
- 지속가능성 관점의 잠재 모델: 지자체·복지관 대상 지역 아동 체력 현황 리포트 제공(B2G), 아동 체육·교구 업체 제휴 노출, 사설 체육시설 프로그램 연계 수수료
- 공모전 제출 단계에서는 수익보다 공공적 가치(국민체력100 센터 유입 확대)를 전면에 둠

### 핵심 기능 정의 (MVP는 별도 표시)
- 회원 기능: 학부모 계정, 자녀 프로필 다중 등록(개월수·성별) [MVP]
- 체력 진단 기능: 입력된 측정값을 국민체력100 유아기 공식 기준표에 대입해 항목별 등급·강약 프로파일 산출 [MVP·핵심]
- 센터 연계 기능: 위치 기반 근처 체력인증센터 검색·안내 [MVP]
- 측정 기록 저장 기능: 정식 측정 결과 입력·저장, 언제든 재조회 [MVP]
- 성장 추적 기능: 시계열 기록으로 체력 요소별 추이 시각화. 정식 측정값(실선·강조)과 자가측정 참고값(점선·흐림)을 구분해 함께 표시, 정식 측정 주기가 길어도 추이가 끊기지 않도록 보완 [MVP]
- 활동 추천 기능: 약한 요소별 가정 활동 콘텐츠(공단 운동처방 동영상) 연계 [MVP]
- 지역 참여 인사이트 기능: 진단 결과에 '우리 동네 유아 측정 참여 vs 전국' 통찰 카드 제공, 여력 시 시군구 참여율 지도/랭킹 시각화 1종 추가 [MVP·통찰 카드 우선 / 시각화는 최종 레이어, 시간 부족 시 카드로 축소]
- 간이 자가측정 기능: 집에서 가능한 종목 입력, '참고용' 명시 (2차 확장 또는 최소 구현. 단, 성장 추적의 참고값 공급원으로 연계)

### 유저 채널(환경)
- 주 사용자: 유아기(만 4~6세, 48~83개월) 자녀를 둔 학부모
- 기기 환경: 모바일 우선(스마트폰). 학부모가 집·외부에서 수시로 입력·조회
- 형태: 반응형 웹 (공모 시제품 개발 부담 최소화, '시제품 구현 완료' 요건 충족 목적)
- 사용 맥락: 집에서 자가측정(참고용) → 센터 방문 후 정식 결과 입력 → 주기적으로 추이 확인하며 재방문

### 사용 공공데이터

#### 필수요건 충족 데이터 (공공데이터포털 등록 정식 오픈 API)
공모 필수요건(공공데이터포털·문화빅데이터플랫폼 등록 공단 데이터 1건 이상 활용)은 아래 2·3번 정식 오픈 API로 충족한다. 1번 기준표는 데이터셋이 아닌 '고시 기반 판정 근거'로 포지셔닝하여 별도 카운트하지 않는다.

#### 1. 국민체력100 유아기 체력인증 기준표 (진단의 근간 / 판정 근거)
- 성격: 데이터셋이 아닌 문화체육관광부 고시 제2025-0027호 기반 공인 판정 기준 (필수요건 카운트에서 제외, 진단 로직의 근거로만 사용)
- 출처: 국민체력100 인증기준 페이지 (https://nfa.kspo.or.kr/reserve/0/selectMeasureGradeItemListByAgeSe.kspo)
- 내용: 유아기(48~83개월) 성별·연령별 등급 경계값. 항목은 심폐지구력(10m 왕복오래달리기), 근력(상대악력), 근지구력(윗몸말아올리기), 유연성(앉아윗몸앞으로굽히기), 민첩성(5m×4 왕복달리기), 순발력(제자리멀리뛰기), 협응력(3×3 버튼누르기), 신체조성(BMI)
- 판정 규칙 (원문 확인 완료, 등급별 평가 항목 집합이 다름):
  - 열매: 건강체력 4개(심폐지구력·근력·근지구력·유연성) + 운동체력 3개(민첩성·순발력·협응력) = 7개 항목 전부가 열매 경계값 충족
  - 꽃: 동일 7개 항목이 꽃 경계값 충족
  - 새싹: 건강체력 4개 + BMI = 5개 항목으로 판정 (운동체력 3개는 평가하지 않음. BMI는 상한선 방식, 예: <17.5)
  - 씨앗: 열매·꽃·새싹 어디에도 미달 시 부여
  - 유아기에는 성인·어르신 같은 조건부 혼합 로직(특정 항목만 충족 시 상위등급)이 없어 단순·명확하게 재현 가능
- 구현 시 주의: 등급마다 필요한 항목 집합이 다르므로, 판정 엔진은 등급별로 평가 항목 키 배열(required_items)을 분리 정의해야 함. 새싹 판정에는 운동체력(순발력·민첩성·협응력) 값이 관여하지 않으므로, 해당 값이 없어도 새싹 판정이 가능해야 함 ("값 없음 → 판정 불가" 버그 방지)
- 데이터 처리: 고시 기반 정적 데이터(수년 단위 개정)이므로 실시간 조회 불필요. 기준표를 1회 파싱해 성별×연령(개월)×등급×항목 구조의 JSON으로 정규화 후 내부 적재. 항목별 방향성(값이 클수록 우수 / 시간·BMI는 작을수록 우수)과 등급별 required_items를 함께 반영
- 운영 고려: 고시 개정 시 재파싱·갱신 절차 필요 (시제품 단계에서는 고려사항으로만 명시)
- 비고: 백분위 대신 국가 공인 등급 기준을 그대로 사용해 데이터 정합성 확보

#### 2. 국민체력100 체력인증센터 측정건수·위치 정보 [필수요건 충족]
- 출처: 국민체육진흥공단 / 공공데이터포털 (Open API, data.go.kr/data/15114286)
- 내용: 전국 체력인증센터명, 기본주소, 위치정보, 측정건수
- 활용: (a) 위치 기반 근처 센터 검색·안내, 정식 측정 연계 (b) 지역별 측정건수를 전국과 비교해 '우리 동네 참여 인사이트' 도출
- 확인 필요: 측정건수가 시군구 단위로 구분되는지, 센터 단위만 제공되는지 실제 API 응답으로 확인. 센터 단위만 제공 시 센터 주소로 지역을 역산해 집계

#### 3. 국민체력100 동영상 정보 [필수요건 충족]
- 출처: 국민체육진흥공단 / 공공데이터포털 (Open API, data.go.kr/data/15108846)
- 사용 엔드포인트: `TODZ_VDO_TRNG_GUIDE_I` (`script/kspo_responses/01_trng_guide.json`)
- 내용: 연령대별·체력요인별·운동부위별 운동처방 가이드 동영상 목록
- 활용: 약한 체력 요소별 가정 활동 추천 콘텐츠로 연계
